## Reglas funcionales (RF-23 a RF-28)

- Un episodio permite multiples evaluaciones: INICIAL, INTERMEDIA y ALTA.
- Solo se permite una evaluacion INICIAL por episodio.
- Solo se permite una evaluacion ALTA por episodio.
- Para registrar INTERMEDIA o ALTA debe existir una evaluacion INICIAL.
- No se pueden registrar evaluaciones en episodios cerrados.
- El sistema calcula automaticamente:
  - IMC
  - porcentaje e interpretacion por escala funcional
  - puntaje funcional promedio
  - interpretacion funcional global

## Catalogos usados

### tipoEvaluacion
- INICIAL
- INTERMEDIA
- ALTA

### resultado en pruebasESpeciales
- POSITIVO
- NEGATIVO
- DUDOSO

## Validaciones relevantes del request

- `tallaCm`: entre 30.0 y 250.0
- `pesoKg`: entre 1.0 y 500.0
- `eva`: entre 0 y 10
- `fuerzaMuscular[].puntajeDaniels`: entre 0 y 5
- `pruebasEspeciales[].resultado`: POSITIVO | NEGATIVO | DUDOSO
- `goniometria`, `fuerzaMuscular`, `escalasFuncionales`, `pruebasEspeciales`: no vacios

## Errores frecuentes (400)

- `Debe registrarse primero la evaluacion inicial del episodio.`
- `El episodio ya tiene una evaluacion inicial registrada.`
- `El episodio ya tiene una evaluacion de alta registrada.`
- `No se pueden registrar evaluaciones en un episodio cerrado.`
- `No existe evaluacion inicial para este episodio.`
- `Evaluacion no encontrada: {evaluacionId} para episodio: {episodioId}`

## Registro de problemas (RF-29)

### POST /episodios-clinicos/{episodioId}/problemas
Registra un nuevo problema en el episodio clínico.

Request:
```json
{
  "descripcion": "Dolor lumbar persistente",
  "codigoCie10": "M54.5",
  "estado": "ACTIVO"
}
```

Response (200):
```json
{
  "id": 1,
  "numeroSecuencial": 1,
  "descripcion": "Dolor lumbar persistente",
  "codigoCie10": "M54.5",
  "estado": "ACTIVO"
}
```

### GET /episodios-clinicos/{episodioId}/problemas
Lista todos los problemas registrados en el episodio clínico.

Response (200):
```json
[
  {
    "id": 1,
    "numeroSecuencial": 1,
    "descripcion": "Dolor lumbar persistente",
    "codigoCie10": "M54.5",
    "estado": "ACTIVO"
  },
  {
    "id": 2,
    "numeroSecuencial": 2,
    "descripcion": "Ciática crónica",
    "codigoCie10": "M54.3",
    "estado": "CRONICO"
  }

```

### Estados posibles
- ACTIVO
- CRONICO
- RESUELTO
- DESCARTADO

### Reglas
- El número secuencial se asigna automáticamente por episodio.
- Solo fisioterapeuta puede registrar problemas.
- La lista es visible al inicio de la HC.

### Casos de uso
Este endpoint es útil para:
- **Pantalla de resumen del episodio**: Mostrar toda la información en un solo panel
- **Frontend dashboard**: Visualizar estado completo del paciente en su episodio actual
- **Verificar si existen**: Evaluar rápidamente si hay problemas, planes o evaluaciones
- **Auditoría**: Obtener historial completo del episodio en una sola consulta
